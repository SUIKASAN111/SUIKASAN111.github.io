指针类型

Type of Pointer Means Nothing！，它既不改变其中的地址，也不会决定该地址存着的数据类型。唯一的差别就是不同指针类型在进行+1操作时候步进的bytes不一样。

引用的本质是指针常量

引用就是一种很典型的语法糖(Syntax Sugar)

int* const ptr;//指针常量——指针是个常量，即指向无法改变
const int* ptr;//常量指针——指针指向的是个常量，即指向的值无法改变，但是指向可以改变

Struct VS Class

技术上说，基本上没区别😂，但是实际使用又有区别
struct中成员默认权限为public，class中成员默认权限为private
struct在C++中存在的唯一理由就是维持其对于C的兼容性

实际使用中没有绝对的用法。但是可以考虑使用struct定义Plain Old Data(POD)，而需要继承时候使用class。只将结构体简单地用作数据结构的定义,而需要实现各种功能时候使用类。But, it’s up to you after all.

Static in C++

类外static：在link阶段是局部的，static修饰的内容只对定义它的translation unit可见。
类内static：static修饰的这部分内存有所有该类的实例共享。普通字段每个实例都会进行实例化，而静态字段只实例化一次而由所有实例对象共享。可以理解为，普通字段属于每个实例对象，而静态对象属于这个类。静态变量需要在类外定义。
静态方法道理类似。静态方法只能访问静态成员。非静态方法会获得所属实例的指针(this)作为参数。
对struct同理
Local Static
如函数中的static变量，只会再第一次调用时创建一次，后面调用不会再创建。

tips

如果在头文件定义了static变量，而在两个cpp文件中包含了头文件，那么两个文件会分别各自定义一个静态变量，因为头文件内容被拷贝过去了。

善用static。事实上，在不希望变量全局可见时应该尽量static。不然linker在会所有翻译单元中总惦记着这个全局变量，这常常导致奇奇怪怪的错误。划重点，尽量让全局函数和变量static，除非必须要用在其他翻译单元中。
类比想想看为什么在类中总是能用private就用private，因为全局可见的特性本身就和封装的思想在一定程度上相背。

extern关键词

修饰变量，表示该变量定义于其他Translation Unit, Linker在链接时候会从其他翻译单元找定义。
修饰常量，表示该常量有外部链接，可被其他翻译单元发现。常量默认只会内部链接，不可被其他翻译单元发现。

Char和String剖析

Const char*字符串后会以一个byte的0(HEX00)，称为空终止符(Null Termination Character)，以表示字符串的结束。

char name1[8] = { 'S','u','i','k','a','s','a','n' };//没有空终止符，打印时不知道何时结束，会一直打印直到遇到00
char name2[9] = { 'S','u','i','k','a','s','a','n', 0 };//'\0'也行
//下面两个打印出来都会是“Sui”
const char* name1 = "Sui\0kasan";
std::string name3 = "Sui\0kasan";

std::string本质上是basic_string模板类以char作为模板参数时的实例，这称为模板特化(Template Specialization)

成员初始化列表

在构造函数中初始化子类，会先实例化一个新的将其赋值给父类实例化时创建的对象之后将其丢弃。而使用成员初始化列表则不会另外实例化一个对象，减少性能浪费。

三目运算符

三目运算符是if-else语句的语法糖。

if (s_Level > 5) {
		s_Speed = 10;
	}
	else {
		s_Speed = 5;
	}

	s_Speed = s_Level > 5 ? 10 : 5;

New关键字

使用New声明变量，申请空闲内存时候会使用空闲列表(Free List)。

智能指针Smart Pointers

**std::unique_ptr:**无法复制
**std::shared_ptr:**引用计数方法，可被复制，每复制一个都增加一次引用计数，所有引用都释放后，引用计数为0，指向的内容才会被释放
**std::weak_ptr:**和shared_ptr类似，但是当把值复制给weak_ptr时，引用计数不会增加

Always pass your objects by const reference

std::vector的拷贝

先在main中创建变量，然后拷贝进入vector中。当vector不够大时，会创建新的vector，然后将原来的元素拷贝至新的vector中。这之中存在许多可以优化的地方。

dynamic_cast原理
